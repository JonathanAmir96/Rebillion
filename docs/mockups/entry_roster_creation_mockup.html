Rebillion — Entry Flow Mock-up: Title, Roster & Character Creation

Three stacked screens at the locked render base 640×360, each shown here at integer 2× — a 1280×720 stage on solid ink, exactly the letterbox law in ART_BIBLE.yaml pixel.resolution_policy / DISPLAY.md §2. Frame/font/button-state vocabulary is UI_ART_SPEC.md's (frame_window, frame_input, frame_button, frame_dialog all appear below); monospace stands in for the bitmap font "GameSans-Pixel". Flow and copy are ACCOUNT.md §1–§3; appearance parts, z-order and swatch hexes are CHARACTER_COMPOSITING.md and ART_BIBLE.yaml amendment AB-002 (real values, not placeholders). Wireframe fidelity only — flat rects/swatches/simple silhouettes, no AI-generated art. Gold numbered pins are design annotations, not game UI.

1 · Title screen (press-enter gate)

On launch the player sees the wordmark and a press-to-continue prompt — see annotation 4 for why this is a title/splash gate rather than a login form in the current build.

1 Wordmark — blocky logotype; monospace stands in for the bitmap font "GameSans-Pixel" UI_ART_SPEC.md Font
2 Press-Enter gate → advances straight to the roster screen ACCOUNT.md §1
3 display_mode hint: Alt+Enter toggle, fullscreen is default DISPLAY.md §1, §3
4 Interim solo build has no credential/login step — §3 (handle+password, session tokens) is dormant until a server exists; title/splash gate only, not a login form ACCOUNTS_AUTH.md §7
5 Locked 640×360 render base, integer 2× shown here; true letterbox remainder is solid ink ART_BIBLE.yaml pixel.resolution_policy, DISPLAY.md §2

Sky/hill art is flavor filler (Emberfoot dusk motif) — not canon key art. The brief calls this screen "title/login"; per ACCOUNTS_AUTH.md §7 the interim-solo scope in play right now has no login UI at all (§2.3–§6 are all dormant), so this stage is the title/splash gate only — see annotation 4.

2 · Character roster (up to 4 slots)

Landing screen after the title gate. Two filled example slots, two empty. Selecting a character enters the world; there is no gameplay outside a selected character.

1 Slot card: nickname / Lv / job display name / composited idle sprite ACCOUNT.md §1
2 Empty slot → Create ACCOUNT.md §1
3 Selected slot: emphasis outline + corner ticks (frame_window ornate-corner motif, shown once for clarity) UI_ART_SPEC.md
4 Enter World (frame_button, primary/emphasis): selecting a character enters the world; no gameplay outside a selected character ACCOUNT.md §1
5 Delete (per filled slot, frame_button icon-only): explicit type-the-nickname confirmation, destructive, no undo ACCOUNT.md §1
6 Delete dialog is frame_dialog (earth trim) over a screen dimmer; confirming frees the slot and releases the nickname back into the pool ACCOUNT.md §1; ACCOUNTS_AUTH.md §4.1/§5
7 4-slot quota, single source of truth ACCOUNTS_AUTH.md §2.2

Card sprites are flat wireframe silhouettes (no AI art) standing in for the real composited idle sprite — see the layered live preview on the creation screen (§3) for the actual base → face → hair composite per CHARACTER_COMPOSITING.md. "Ember" shows a 1st-job display name (Bulwark, earned at Lv 8); a fresh slot always starts Novice.

3 · Character creation (3 linear steps)

Nickname → Appearance → Confirm, cancellable at any point back to the roster. Appearance is shown expanded since it carries the richest widgetry (pickers + live preview); nickname is recapped top-left, confirm summarized in the footer bar.

1 Step rail: Nickname → Appearance → Confirm, linear + cancellable at any point ACCOUNT.md §2
2 Nickname is a frame_input (sunken) field; Check Name is a frame_button (raised, normal state); both possible answers shown — available (good) / taken (warn) ACCOUNT.md §2.1
3 Hair — 12 style_hair_NN options (6 shown + count) CHARACTER_COMPOSITING.md §7, docs/ID_REGISTRY.md
4 Face — 8 style_face_NN options (6 shown + count); abstract at 32px, no separate mouth layer CHARACTER_COMPOSITING.md §3.1, §7
5 Hair color — 6 style_haircolor_NN swatches, real 3-step ramps; hair_violet is exempt from the reserve-arcane-for-rare/elite rule ART_BIBLE.yaml amendment AB-002
6 Skin — 5 style_skin_NN swatches, real 4-step ramps ART_BIBLE.yaml amendment AB-002
7 Live preview composites as picks change; exploded stack shows z-order base(z1) → face(z6) → hair(z7); base body fixed style_base_00 CHARACTER_COMPOSITING.md §2, §7
8 Confirm mints the record — always Novice, spawns at map_001 Emberfoot Village; disabled until the name check returns available (frame_button disabled state) ACCOUNT.md §2.2–§2.3, §3; JOBS.md; WORLD_PLAN.md
9 Picks must fall inside their docs/ID_REGISTRY.md reserved block (hair 12/40, face 8/20, skin 5/10, haircolor 6/10) ID_REGISTRY.md "Appearance styles"

Cancel returns to the roster; any nickname hold from step 1 is released — availability is session-scoped, never a permanent hold. Swatch/thumbnail counts shown are illustrative subsets to keep the wireframe legible; the real catalog is 50_content/styles/style_catalog.yaml ACCOUNT.md §2; CHARACTER_COMPOSITING.md §8
